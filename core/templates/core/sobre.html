Objetivo do projeto

Este repositório é um exemplo didático de Progressive Web App com Django: o mesmo código serve no navegador e pode ser instalado na tela inicial, com manifest, service worker e página offline.

O “contexto de negócio” é propositalmente simples: até três tarefas por dia, histórico e revisão — o bastante para CRUD, autenticação e fluxo real, sem esconder o que importa aqui: boas práticas de PWA.

O que você pode demonstrar com este app

  • Web App Manifest — nome, ícones, display: standalone, atalhos.
  • Service Worker — cache de estáticos e fallback quando não há rede.
  • Instalação — prompt / “Adicionar à tela inicial” e banner no template base.
  • Backend Django — usuários, modelos e formulários alimentando o front.

Limitações de propósito

O escopo é intencionalmente pequeno: foco em fluxo de tarefas diário e recursos de PWA. Trata-se de um projeto de portfólio e aprendizado, não de um produto comercial pronto.
